\begin{DoxyParagraph}{}

\begin{DoxyCode}{0}
\DoxyCodeLine{export CXX=<path-\/to-\/your-\/C++-\/compiler>}
\DoxyCodeLine{git clone -\/-\/recursive git@github.com:Pressio/pressio4py.git \# or https://github.com/Pressio/pressio4py.git}
\DoxyCodeLine{pip install ./pressio4py}
\end{DoxyCode}

\end{DoxyParagraph}
To make sure everything worked, run the \href{https://github.com/Pressio/pressio4py/tree/master/regression_tests}{\texttt{ regression tests}} as follows\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{cd pressio4py}
\DoxyCodeLine{pytest -\/s}
\end{DoxyCode}


If you get an import error it might be that the version of {\ttfamily pytest} you are using is not compatible with the {\ttfamily pip} command you used to install. Make sure you use Python commands from the {\bfseries{same}} distribution.\hypertarget{index_autotoc_md53}{}\doxysection{Learn how to interface your app}\label{index_autotoc_md53}


\begin{DoxyParagraph}{}
To enable pressio4py to communicate with your app, you might have to prepare some glue code in the form of an adapter class. You can find the details \href{./md_pages_prepare_your_app.html}{\texttt{ here}}. Note that this step only needs to be done once\+: the same interface class can then be used to run all the R\+O\+Ms in pressio4py.
\end{DoxyParagraph}
